## 新增需求

### 需求：创建绑定关系

系统 SHALL 允许管理员创建飞书机器人和 Dify 应用之间的绑定。

#### 场景：创建有效绑定
- **WHEN** 管理员创建绑定，指定 feishu_bot_id 和 dify_app_id
- **THEN** 系统验证两个 ID 都存在且已启用
- **THEN** 系统验证不存在重复绑定
- **THEN** 系统创建绑定，记录优先级（默认 0）
- **THEN** 系统返回创建的绑定记录

#### 场景：防止重复绑定
- **WHEN** 管理员尝试创建已存在的绑定
- **THEN** 系统返回错误"绑定已存在"

### 需求：绑定路由

系统 SHALL 根据绑定关系将飞书消息路由到对应的 Dify 应用。

#### 场景：基于优先级的路由
- **WHEN** 飞书消息到达某个机器人
- **THEN** 系统查找该机器人所有已启用的绑定
- **THEN** 系统按优先级降序排序绑定
- **THEN** 系统返回优先级最高的绑定所对应的 Dify 应用

#### 场景：未找到绑定
- **WHEN** 飞书消息到达一个没有绑定的机器人
- **THEN** 系统返回错误"该机器人未绑定任何 Dify 应用"

#### 场景：Dify 应用已禁用
- **WHEN** 绑定的 Dify 应用已被禁用
- **THEN** 系统跳过该绑定
- **THEN** 系统查找下一个可用的绑定

### 需求：绑定管理

系统 SHALL 允许对绑定关系进行增删改查操作。

#### 场景：更新绑定优先级
- **WHEN** 管理员更新绑定的优先级
- **THEN** 系统保存新的优先级值
- **THEN** 系统清除该机器人的 Redis 缓存

#### 场景：启用/禁用绑定
- **WHEN** 管理员切换绑定的启用状态
- **THEN** 系统更新启用标志
- **THEN** 系统清除 Redis 缓存

#### 场景：删除绑定
- **WHEN** 管理员删除一个绑定
- **THEN** 系统移除绑定记录
- **THEN** 系统清除 Redis 缓存

#### 场景：列出绑定（包含详情）
- **WHEN** 管理员请求 GET /api/bindings
- **THEN** 系统返回所有绑定，记录中包含机器人名称和 Dify 应用名称
