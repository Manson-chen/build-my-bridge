## 为什么需要这个项目

企业需要将飞书机器人与 Dify（私有部署的 AI 应用平台）进行快速集成，但目前缺乏一个简便的解决方案。用户希望只需配置飞书机器人的 AppID/AppSecret 和对应的 Dify 应用，即可实现飞书与 Dify Chatflow/Workflow 的双向打通，无需复杂开发。

## 将会改变什么

- 创建完整的 BuildMyBridge 系统，实现飞书机器人与 Dify 的集成
- 提供 Web 管理界面，用于配置机器人和 Dify 应用信息
- 支持飞书 OAuth 扫码登录认证
- 实现消息处理流程：接收飞书消息 → 调用 Dify API → 返回响应
- 支持一对多绑定：一个飞书机器人绑定一个 Dify 应用，一个 Dify 应用可被多个机器人使用
- 支持灵活配置每个绑定关系的会话超时时间和错误信息展示策略

## 功能范围

### 新增功能

- **飞书集成能力**：WebSocket 长连接接收、消息解析、签名验证、消息推送、OAuth 登录
- **Dify 集成能力**：支持 Chatflow 和 Workflow 两种应用类型的 API 调用
- **机器人绑定能力**：一对多绑定关系管理（一个 Bot 绑定一个 Dify App，一个 App 可被多个 Bot 使用）
- **Web 管理界面**：React 18 + Ant Design SPA 管理后台，支持 Bot、Dify 应用的增删改查
- **消息日志能力**：消息往来日志记录、查看、筛选和统计分析
- **会话状态管理**：基于 Redis 的会话状态，支持上下文对话（超时时间可配置）
- **绑定配置能力**：支持为每个绑定关系配置会话超时时间和错误信息展示策略
