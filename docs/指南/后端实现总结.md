## 🔒 安全设计

### 认证流程

1. **JWT 生成** - OAuth 成功后生成 JWT
2. **JWT 验证** - 每次请求通过 JwtAuthenticationFilter 验证
3. **Token 存储** - Redis 缓存用户 session（可选）
4. **Token 过期** - 24 小时后需要重新 OAuth

### 加密策略

- **AppSecret** - MD5 加密存储（生产环境建议 AES）
- **JWT 密钥** - 环境变量存储，不上传代码
- **数据库密码** - 环境变量存储

## 📊 数据库架构

### 表结构

```sql
-- 用户表
CREATE TABLE `user` (
  `id` BIGINT PRIMARY KEY AUTO_INCREMENT,
  `username` VARCHAR(255) NOT NULL UNIQUE,
  `feishu_open_id` VARCHAR(255),
  `created_at` TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 机器人表
CREATE TABLE `bot_account` (
  `id` BIGINT PRIMARY KEY AUTO_INCREMENT,
  `bot_type` VARCHAR(50) NOT NULL,
  `bot_name` VARCHAR(255) NOT NULL,
  `bot_config` JSON NOT NULL,
  `enabled` BOOLEAN DEFAULT true,
  `created_at` TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- AI 应用表
CREATE TABLE `ai_app` (
  `id` BIGINT PRIMARY KEY AUTO_INCREMENT,
  `app_type` VARCHAR(50) NOT NULL,
  `app_name` VARCHAR(255) NOT NULL,
  `app_config` JSON NOT NULL,
  `enabled` BOOLEAN DEFAULT true,
  `created_at` TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 绑定表
CREATE TABLE `subscription` (
  `id` BIGINT PRIMARY KEY AUTO_INCREMENT,
  `bot_account_id` BIGINT NOT NULL,
  `ai_app_id` BIGINT NOT NULL,
  `session_timeout_minutes` INT DEFAULT 120,
  `show_error_to_user` BOOLEAN DEFAULT false,
  `routing_rules` JSON,
  `enabled` BOOLEAN DEFAULT true,
  `created_at` TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (`bot_account_id`) REFERENCES `bot_account`(`id`),
  FOREIGN KEY (`ai_app_id`) REFERENCES `ai_app`(`id`)
);

-- 消息日志表
CREATE TABLE `message_log` (
  `id` BIGINT PRIMARY KEY AUTO_INCREMENT,
  `bot_account_id` BIGINT NOT NULL,
  `ai_app_id` BIGINT NOT NULL,
  `message_content` TEXT,
  `response_content` TEXT,
  `status` VARCHAR(50),
  `error_message` TEXT,
  `created_at` TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  INDEX (`bot_account_id`),
  INDEX (`ai_app_id`),
  INDEX (`created_at`)
);
```

## 🔄 关键业务流程

### OAuth 认证流程

```
1. 用户访问前端 → 前端检测未认证
2. 前端调用 GET /auth/login
3. 后端返回飞书授权 URL
4. 用户在飞书中扫码授权
5. 飞书重定向 → /auth/callback?code=xxx
6. 后端交换 code 获取 access_token
7. 后端获取用户信息
8. 后端保存用户到数据库
9. 后端生成 JWT Token
10. 前端保存 JWT，后续请求使用
```

### 消息处理流程（待实现）

```
1. 用户在飞书发送消息
2. 飞书服务器 WebSocket 事件
3. FeishuEventHandler 接收
4. 验证消息签名（3秒内）
5. 快速返回 challenge 确认
6. 异步入队 Redis
7. MessageHandlerService 处理
8. 调用 Dify API
9. 获取 AI 回复
10. FeishuMessageSender 发送回复
11. 记录到 MessageLog
```

## 📝 开发建议

1. **新增 API 时**
   - 在 Controller 中添加 `@Tag` 和 `@Operation` 注解
   - 在 DTO 中添加 `@Schema` 注解
   - 遵循统一的 `ApiResponse` 格式

2. **新增 Service 时**
   - 使用 MyBatis-Plus 提供的 CRUD 方法
   - 业务逻辑集中在 Service 层
   - 异步操作使用 `@Async` 或 Redis Queue

3. **新增数据表时**
   - 创建 Entity 类并使用 MyBatis-Plus 注解
   - 创建 Repository 接口
   - 添加必要的索引和外键

4. **测试**
   - 使用 Knife4j 在线测试 API
   - 单元测试：关键业务逻辑
   - 集成测试：DAO 和 Service 层
